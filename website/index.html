[ 02 / OVERVIEW ]

A SPATIAL
COCKPIT FOR
ANY ROBOT
WITH A CAMERA.

Mount an Intel RealSense D435i on a robot. Run the small Python server. Put on your Vision Pro. Locomotion gives you a low-latency video feed in immersive space and turns your headset into a control surface — with a simulation mode for when the hardware isn't on your desk.

  • 01 REAL-TIME VIDEO over WebRTC. Color, colorized depth, or infrared.
  • 02 FIVE INPUT PROTOCOLS, switchable from the landing screen.
  • 03 SIMULATION MODE for testing motion without the robot.
  • 04 OPEN SOURCE. MIT-licensed. Fork it. Extend it.

[ 03 / CONTROL ]

PICK YOUR INPUT.
SWAP IT LIVE.

01

JOYSTICK 2D

A familiar planar joystick window. Drive on the ground plane with linear and angular velocity. The default for quick demos and bring-up.

02

JOYSTICK 3D

A spatial joystick that lives inside the immersive scene. Push it through 3D space to command motion in three axes.

03

GESTURE

Hand pose drives the robot. Pinches, drags, and orientation map straight to velocity commands sent over the data channel.

04

HAND TRACKING

Full skeletal hand input for finer-grained control — designed for tasks where hand shape carries meaning, not just position.

05

DEVICE TRACKING

Your head is the controller. Headset position and orientation drive the robot, freeing your hands entirely.

06

BRING
YOUR OWN

Each protocol is a swappable module. Add one by conforming to the input view-model contract and registering it in the landing picker.

[ 04 / HOW IT WORKS ]

THREE PIECES.
ONE WIRE.

01

ROBOT

A Python server reads from the RealSense D435i, encodes the selected stream, and opens a WebRTC peer connection. Falls back to a regular webcam if no RealSense is attached.

→

02

WIRE

Video flows one way. Commands flow the other. A bidirectional data channel carries control input and stream-switch messages — stream:color, stream:depth, stream:infrared.

→

03

HEADSET

The visionOS client renders the remote track in an immersive scene, captures input from the chosen protocol, and pushes velocity commands back to the robot in real time.

[ 05 / SIMULATION ]

NO ROBOT?
NO PROBLEM.

Locomotion ships with a simulation mode that runs the same input pipeline against a virtual robot in an immersive scene. Tune a control protocol, validate a gesture, or demo the system — no hardware required.
